#### 4.2.2 舆情监测

|   |   |   |   |   |   |
|---|---|---|---|---|---|
|功能名称|功能说明|前置条件|业务流程|   |异常处理|
|用户动作|系统响应|
|文章监测|用户可以通过发布时间、信息属性、文章来源、地域、关键字，来查询以获取的舆情信息数据|1. 用户处于登录状态|1. 选择时间、属性、地域等各种参数<br>    <br>2. 点击“查询”按钮|2. 基于用户输入信息，在数据库中模糊匹配查询(对于关键词，先对special_alert_setting和special_report_setting中的所有项的关键词进行模糊匹配，然后基于查询到的alert_id和special_report_id去查询它们在news_data中对应的数据)||
|统计图表|用户可以在进行文章监测操作后，查看统计图表，以直观地感受舆情情况|1. 用户处于登录状态<br>    <br>2. 用户已进行文章监测且仍处于当前页面|1. 点击"统计图表"按钮|1. 展示统计图表||
|检索舆情报告信息|用户可以选择特定舆情报告，查看其对应的舆情信息|1. 用户处于登录状态<br>    <br>2. 存在舆情报告|1. 选择可查看的舆情报告<br>    <br>2. 点击“查询”按钮|1. 展示用户可见的所有舆情报告<br>    <br>2. 展示该舆情报告对应的所有消息||
