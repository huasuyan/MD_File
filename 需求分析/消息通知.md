#### 4.2.7 消息通知

| 功能名称          | 功能说明                        | 前置条件                                | 业务流程                                        |                                                                                                                               | 异常处理 |
| ------------- | --------------------------- | ----------------------------------- | ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- | ---- |
|               |                             |                                     | 用户动作                                        | 系统响应                                                                                                                          |      |
| 预警专题站内通知      | 预警专题符合发送预警通知条件时，能发送站内信，提醒用户 | 1. 用户已登录<br>    <br>2. 此时处于可通知时间内   | 无                                           | 1. 预警专题启用中，获取到的预警记录达到阈值，判断是否处于可通知状态，如果可通知，则向前端发送websocket通知，并将预警通知写入数据库<br>    <br>2. 前端收到websocket通知，右下角浮窗通知用户，并立刻调用接口刷新消息记录 |      |
| 预警通知历史记录与已读标记 | 用户可以在“我的消息”中查看未读与已读的预警通知消息  | 1. 用户已登录                            | 1. 点击“我的消息”<br>    <br>2. 勾选未读消息，点击“批量标记已读” | 1. 查询后端数据库，获取当前用户的预警通知并展示<br>    <br>2. 前端调用后端接口，将对应预警通知标记为已读                                                                 |      |
| 手动刷新消息        | 用户可以在“我的消息”中手动刷新，获取最新消息     | 1. 用户已登录<br>    <br>2. 用户处于“我的消息”页面 | 1. 点击“刷新”按钮                                 | 1. 立刻调用后端接口，获取一次预警通知查询结果                                                                                                      |      |
