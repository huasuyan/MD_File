# 数据库设计

user表

| 字段               | 数据类型         | 约束                                                  | 说明          |
| ---------------- | ------------ | --------------------------------------------------- | ----------- |
| user_id          | bigint       | 主键、自增、非空                                            | 用户唯一 ID     |
| username         | varchar(50)  | 非空、唯一                                               | 用户名（账号）     |
| password         | VARCHAR(200) | 非空                                                  | 加密后的密码      |
| phone            | VARCHAR(20)  | 唯一                                                  | 手机号（用于短信登录） |
| create_time      | DATETIME     | 非空、默认 CURRENT_TIMESTAMP                             | 账号创建时间      |
| update_time      | DATETIME     | 非空、默认 CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 最后修改时间      |
| last_operation   | varchar(100) | 可为空                                                 | 最后一次操作      |
| last_active_time | datetime     | 可为空                                                 | 最后一次操作时间    |

舆情报告专题设置表

| 字段                | 数据类型         | 约束                                 | 说明                                 |

| create_user_id    | int          | NOT NULL、外键关联User表                 | 创建人ID                              |

| monitor_region    | varchar(300) | 可为空                                | 监测地域                               |
| report_type       | int          | 非空                                 | 报告类型：<br><br>1：即时报告 2：定时报告         |

| status_enabled    | int          | NOT NULL 默认: 0                     | 启用状态：<br><br>1 = 启用，<br><br>0 = 停用 |

预警专题表

special_alert_setting

|字段|数据类型|约束|说明|
|---|---|---|---|
|alert_id|int|主键、非空|预警专题ID|
|user_id|bigint|外键、非空|用户ID|
|alert_name|varchar(50)|非空|预警专题名称|
|trigger_state|int|非空|0：已停止 1：已启用|
|target_source|varchar(20)|非空|数据源: xinhuanet:新华网<br><br>integration：综合|
|key_word|json|非空|预警词组|
|params|json|默认为null|可变参数|
|frequency|int|非空|表示每隔多少分钟执行|
|alert_trigger|int|默认为null<br><br>frequency为2时，alert_trigger非空，默认为1|预警阈值<br><br>表示每有多少条舆情信息发送预警消息，为null时表示实时预警|
|time_range|json|默认为null|预警时间范围，如<br><br>{<br><br>"weekdays":[0, 1, 1, 1, 1, 1, 0],<br><br>"time":{<br><br>  "start": "06:00", "end": "22:00"<br><br>  }<br><br>}<br><br>，表示每周一~周五6点~22点预警<br><br>默认为null，此时任何时间都可以预警|
|alert_method|int|非空，默认1|预警方式<br><br>0：全部<br><br>1：站内信<br><br>2：邮箱|
|dedup_enable|int|非空，默认1|预警重复信息去重<br><br>0：不去重<br><br>1：去重|
|state|int|非空|-1：监测失败，等待下一次执行 0：等待下一次执行 1：爬取数据中 2：数据清洗中 3：数据保存中|
|latest_news_time|DATETIME|默认为Null|数据库中publishTime最接近现在的时间|
|create_time|DATETIME|非空|任务创建时间|
|pending_count|int|默认为0|待发预警的累计新增舆情数|
|last_trigger_time|datetime|默认为Null|上次触发爬虫的时间，用于frequency间隔判断|
